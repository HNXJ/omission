# Neuroscience Writing: arXiv & bioRxiv Templates

Guides for formatting preprints for early dissemination.

## 1. arXiv Two-Column Format
arXiv typically accepts LaTeX-generated PDFs using the `revtex4` or `article` class with `multicol`.

- **Visual Hierarchy**: 
    - Full-width Title and Abstract.
    - Two-column main text for high information density.
    - Large multi-panel figures often span both columns (top/bottom) for clarity.
- **Tone**: Can be slightly more technical or "working paper" style compared to the final journal version.

## 2. bioRxiv Specifications
- **Format**: Prefers a single-column format for the main submission, but two-column is accepted.
- **Linkage**: Essential to include DOIs for all references to facilitate automated cross-linking with later journal versions.
- **Supplemental**: Standard to include a "Materials and Methods" or "Supplemental Information" as a separate PDF or integrated at the end.

## 3. Preprint Best Practices
- **Disclaimer**: Always include a statement: "This is a preprint and has not yet been peer-reviewed."
- **Versioning**: arXiv allows versioning (v1, v2); use this to push updated consensus scores as we refine agent parameters.
- **Code Link**: Prominently feature the GitHub repository (e.g., AAE) in the abstract or first page.
