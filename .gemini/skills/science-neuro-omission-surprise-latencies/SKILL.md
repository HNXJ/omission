# Quantifying Surprise Latencies

The speed of surprise signaling is a critical metric for understanding the hierarchical implementation of Predictive Coding. In our sequential task, we measure the latency of the first significant divergence in firing rate between expected presentations (e.g., P2 in AAAB) and omissions (e.g., X in AXAB).

Hierarchy of Latencies:
We observe a distinct V1-PFC lag. While V1 responds to visual stimuli within 40-60ms, its response to omissions (if any) is significantly delayed or weak. Conversely, PFC and FEF exhibit rapid surprise transients. In many sessions, FEF latencies for omission detection are between 12ms and 41ms relative to the expected stimulus onset (Code 103/105/107).

Calculation Methodology:
Latency is defined as the first of 20 consecutive 1ms bins where the firing rate for the omission condition exceeds the 95th percentile of the baseline (pre-stimulus) activity, or where the AUC (Area Under Curve) between Omission and Stimulus traces significantly deviates from 0.5.

Comparison:
- V1: ~100-150ms (Feedback-driven, low magnitude).
- MT/FST: ~40-60ms (Intermediate).
- FEF/PFC: ~15-45ms (Prediction-driven, high magnitude).

Technical Implementation:
```python
import numpy as np
def calculate_latency(trace_omit, trace_baseline, threshold=3.0):
    # Z-score omission relative to baseline
    z_score = (trace_omit - np.mean(trace_baseline)) / np.std(trace_baseline)
    # Find first crossing
    idx = np.where(z_score > threshold)[0]
    if len(idx) > 0:
        return idx[0] # Latency in ms
    return None
```

Scientific Context:
The fact that higher areas respond faster to omissions than lower areas is a hallmark of top-down prediction error signaling. It confirms that the surprise is generated by an internal clock/model rather than being a delayed sensory response.
