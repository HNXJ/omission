# skill: analysis-nwb-data-availability-report

## When to Use
Use this skill for project-level data auditing. It is essential for:
- Identifying "broken" sessions with missing critical data (e.g., no LFP).
- Quantifying the total yield of units across the entire project.
- Creating a "Stable-Plus" exclusion list based on data quality (FR, SNR, trial presence).
- Validating NWB export integrity after bulk processing.

## What is Input
- **NWB Directory**: Path to the folder containing session `.nwb` files.
- **Modality Tags**: List of sensors to check (e.g., `LFP`, `Spikes`, `Pupil`).

## What is Output
- **Markdown Table**: `DATA_AVAILABILITY_SUMMARY.md` showing a per-session grid of data presence.
- **Yield Summary**: Total count of units and trials across all valid sessions.
- **Incomplete Sessions List**: Log of files failing the minimum data requirements.

## Algorithm / Methodology
1. **File Discovery**: Recursively searches for all `.nwb` files in the source directory.
2. **Sequential Probe**: Opens each file using `pynwb.NWBHDF5IO`.
3. **Metric Extraction**: Checks for the existence of:
    - `acquisition['LFP']`: Local Field Potentials.
    - `units`: Single/Multi-unit activity.
    - `acquisition['EyeX']` / `acquisition['EyeY']`: Oculomotor traces.
    - `intervals['trials']`: Behavioral event metadata.
4. **Aggregation**: Formats findings into a pandas DataFrame and exports as a Markdown table.
