# Behavioral Task Details: MonkeyLogic (.mat) Data

## 🧠 Behavioral Data Structure
- **Behavioral controller**: MonkeyLogic 2.2.
- **Top-Level Variable**: `bhvUni` (an array of trial structures).
- **Core Trial Fields**:
  - `TrialError`: `0` indicates a correct trial (fixation maintained throughout). Any other value (e.g., 1, 2, 3) indicates an error (break of fixation, late start, etc.).
  - `AnalogData.Eye`: A `(nSamples, 2)` array containing the calibrated eye position in **Degrees of Visual Angle (DVA)**.
  - `BehavioralCodes`: Contains two parallel arrays: `CodeNumbers` (event IDs) and `CodeTimes` (timestamps in ms relative to the start of the trial).
  - `TaskObject.Attribute`: Metadata for each stimulus presentation, used to resolve stimulus identity (A, B, or R).

## 👁️ Oculomotor Information
- **Unit Standard**: All eye trajectories in the `.mat` files are pre-calibrated to **DVA** (Degrees of Visual Angle) based on the screen distance (57 cm).
- **Sampling Rate**: Eye position is sampled at **1000 Hz** (1 ms per sample).
- **Fixation Window**: The subject must maintain eye position within a small window (typically ~1-2° radius) centered at `(0, 0)`.
- **Classification**:
  - **Saccades**: Velocity > 30 DVA/s and Amplitude > 1.5°.
  - **Microsaccades**: Amplitude < 1.5° during stable fixation periods.

## 🕒 Timing & Synchronization
- **Fixation Alignment**: The task sequence begins at **Code 10** (Fixation Onset).
- **Stimulus Onsets**:
  - **P1**: Code 101.
  - **P2**: Code 103 (or omission).
  - **P3**: Code 105 (or omission).
  - **P4**: Code 107 (or omission).
- **Reward**: Code 96 indicates the successful delivery of the reward (juice).
- **Standard Trial Window**: 5248ms (from Fixation Onset to end of trial sequence).

## 🔄 Stimulus Identity Mapping
Stimulus identity is resolved by parsing the file paths in `TaskObject.Attribute`:
- **'A.avi'**: Visual Stimulus A (45° grating).
- **'B.avi'**: Visual Stimulus B (135° grating).
- **'R.avi'**: Random stimulus (either A or B, decided per trial).
- **'O' (Empty/Omission)**: Screen remains gray (luminance-matched).

## 🧪 Comparison with NWB (Neurophysiology)
- **Subsetting**: While the `.mat` files contain every trial (Correct + Errors), the NWB (`.npy`) files are a strictly filtered subset of **Correct trials (TrialError == 0)** that also pass additional signal quality checks.
- **Alignment**:
  - **BHV (.mat)**: Usually aligned to Trial Start or Fixation Onset (Code 10).
  - **NWB (.npy)**: Standards use Code 101.0 (P1 Onset) with a 1000ms pre-stimulus buffer.
- **Units**:
  - **BHV**: Degrees of Visual Angle (DVA).
  - **NWB**: Raw signal (Volts) or z-scored activity (SPK/MUAe). For behavioral NWB signals, Z-scoring is required for shape matching with BHV DVA traces.
